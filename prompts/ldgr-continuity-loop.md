# LDGR Continuity Loop

Before doing project work, run:

```sh
ldgr status
```

Use `ldgr context` only when the brief status is insufficient.

Take exactly one pending work item. Start one run with `ldgr run start`, record only durable observations and artifacts, validate the work, and close the run before signing off. Queue a follow-up work item only when new scope is discovered.

For this project, preserve the event-first memory invariant: raw signals produce detected events first; transitions, motifs, static projections, and visualizations are derived views.
